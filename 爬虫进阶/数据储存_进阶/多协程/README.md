# Python 异步爬虫教学示例集

本目录包含多个使用 `asyncio` + `aiohttp` 实现异步网络请求的脚本，以及使用 `requests` + `parsel` 解析小说目录与章节的完整示例。代码从基础异步语法到完整异步爬虫逐步深入，适合教学场景。

## 目录结构

```
.
├── asyncio_模块初使用.py      # asyncio 基础语法与协程演示
├── aiohttp_简单使用.py        # aiohttp 异步请求并保存 HTML
├── 协程实战.py                # 完整异步爬虫：下载整本小说
├── 56283.txt                  # 示例小说目录页 HTML（师叔，你的法宝太不正经了）
├── 57955.txt                  # 示例小说目录页 HTML（斗罗绝世：我霍雨浩，精神之主）
└── 57993.txt                  # 示例小说目录页 HTML（我，枪神！）
```

## 环境依赖

所有脚本需要 Python 3.7+，并安装以下库：

```bash
pip install aiohttp aiofiles requests parsel fake-useragent
```

> 说明：`asyncio` 和 `time` 等为标准库，无需额外安装。

## 文件详解

### 1. `asyncio_模块初使用.py`
**教学目的**：理解 `async` / `await` 基础语法，掌握 `asyncio.gather()` 并发执行协程。

- **第一部分（注释）**：演示 5 个协程并发执行，对比同步 `time.sleep` 与异步 `asyncio.sleep` 的效果。
- **第二部分**：模拟爬虫下载流程，创建 4 个协程任务，并发执行并统计总耗时。

**运行**：
```bash
python asyncio_模块初使用.py
```

**输出示例**：
```
开始执行
开始执行
开始执行
开始执行
www.baidu.com执行完成
www.qq.com执行完成
www.sinaiping.com执行完成
www.163.com执行完成
3.004s   # 实际耗时约 3 秒（并发）
```

**教学要点**：
- 协程函数用 `async def` 定义，调用时返回协程对象，不会立即执行。
- `await` 挂起当前协程，让出控制权给事件循环。
- `asyncio.gather(*tasks)` 并发运行多个协程。
- 异步 I/O 可以大幅提高网络密集型任务的效率。

### 2. `aiohttp_简单使用.py`
**教学目的**：学习使用 `aiohttp` 发起异步 HTTP 请求，并将响应内容保存为本地文件。

- **功能**：依次请求三个小说目录页的 URL，将返回的 HTML 源代码保存为 `.txt` 文件。
- **文件名生成规则**：从 URL 中提取 `list/` 后面的数字作为文件名（例如 `57993.txt`）。

**运行**：
```bash
python aiohttp_简单使用.py
```

**输出**：会在当前目录生成三个文件：`57993.txt`、`56283.txt`、`57955.txt`（内容与您提供的示例文件一致）。

**教学要点**：
- `async with aiohttp.ClientSession() as session` 管理会话。
- `session.get(url)` 是协程，需要 `await`。
- `await response.text()` 异步获取文本内容。
- 异步文件写入建议使用 `aiofiles`（示例中使用了同步写入，教学时可对比效果）。

### 3. `协程实战.py`
**教学目的**：综合运用 `requests`（同步）获取目录列表，再结合 `aiohttp` + `aiofiles`（异步）批量下载整本小说的所有章节。

**核心流程**：
1. **同步部分**（`get_url` 函数）：
   - 使用 `requests` 获取小说目录页。
   - 使用 `parsel` 解析章节链接（兼容 `div#list dl` 和 `ul.chapter` 两种结构）。
   - 返回章节 URL 列表和书名。
2. **异步部分**（`aiodownload` 函数）：
   - 对每个章节 URL 使用 `aiohttp` 发起请求。
   - 解析章节标题和正文（支持两种页面结构）。
   - 使用 `aiofiles` 异步写入 `.txt` 文件，按书名创建文件夹。
3. **主函数**（`main`）：
   - 生成随机 User-Agent。
   - 调用 `get_url` 获取 URL 列表。
   - 创建协程任务列表，并发执行所有章节下载。
   - 统计总耗时。

**运行**：
```bash
python 协程实战.py
```

**输出**：在 `D:\小说\<书名>` 目录下生成每个章节的独立 `.txt` 文件。

**教学要点**：
- 同步与异步混合使用的场景：目录页请求次数少，用同步简单；章节数量多，用异步并发提速。
- `aiofiles` 用于异步文件 I/O，避免阻塞事件循环。
- 异常处理：捕获 `asyncio.TimeoutError` 和通用异常，增强健壮性。
- CSS 选择器兼容不同页面结构，体现实际爬虫的适应性。
- 随机延迟虽然减少但可加入（当前仅同步部分有延迟，异步部分可按需加入 `await asyncio.sleep`）。

**修改提示**：默认下载小说 ID 为 `56896`，您可以更改为其他 ID（如 `56283`）测试。

### 4. HTML 示例文件（`56283.txt`、`57955.txt`、`57993.txt`）
**来源**：使用 `aiohttp_简单使用.py` 或手动保存的小说目录页 HTML。

**用途**：
- 作为解析规则（CSS 选择器）的调试样本。
- 演示不同小说列表的 HTML 结构差异（例如 `div#list dl` vs `ul.chapter`）。
- 可供 `协程实战.py` 离线测试（将 `get_url` 中的请求改为读取本地文件）。

## 教学建议顺序

1. **基础异步**：`asyncio_模块初使用.py` → 理解协程和并发。
2. **异步 HTTP**：`aiohttp_简单使用.py` → 学习用 aiohttp 请求网页。
3. **实战项目**：`协程实战.py` → 完整爬虫，整合目录解析 + 异步下载章节。
4. **对比效率**：可以让学生写一个同步版本（`requests` 循环下载），对比时间差异，直观感受异步的优势。

## 常见问题

- **`aiohttp` 连接超时**：可以在 `session.get()` 中添加 `timeout` 参数，如 `timeout=aiohttp.ClientTimeout(total=10)`。
- **目标网站反爬**：示例网站 `xzmncy.com` 可能随时变更结构或拒绝访问，教学时应强调尊重 `robots.txt` 和合理设置延迟。
- **Windows 路径问题**：`协程实战.py` 中使用了 `D:\小说` 硬编码路径，可改为 `./小说` 或通过 `os.getcwd()` 动态获取。
- **编码问题**：保存文件时使用 `utf-8` 编码，确保中文正确显示。

## 扩展练习

- 修改 `协程实战.py`，将每个章节的正文合并为一个完整的 `.txt` 文件。
- 为异步请求增加重试机制（例如使用 `tenacity` 库）。
- 使用 `asyncio.Semaphore` 控制并发数量，避免对目标服务器造成过大压力。
- 将解析逻辑封装为类，便于复用。
