# Python 数据库操作与异步爬虫教学示例集

本目录包含多个演示 **Python 操作 MySQL 数据库** 以及 **异步爬虫 + 数据库存储** 的脚本。代码从基础连接、单条插入、批量去重插入，到完整的中关村在线笔记本参数异步爬虫，逐步深入，适合作为 Python 数据库编程和异步爬虫的教学材料。

## 目录结构

```
.
├── mysql数据库连接.py           # 基础数据库连接与查询
├── MySQL数据插入.py             # 创建数据库/表，单条数据插入
├── MySQL批量插入和去重.py       # 批量插入 + INSERT IGNORE 去重
└── 中关村笔记本数据.py          # 完整异步爬虫：抓取笔记本参数并存入 MySQL
```

## 环境依赖

所有脚本需要安装 `pymysql`，异步爬虫还需 `aiohttp`、`aiomysql`、`parsel`、`fake-useragent` 等库。

```bash
pip install pymysql aiohttp aiomysql parsel fake-useragent
```

> 注：`asyncio`、`uuid`、`logging`、`random`、`time` 等为 Python 标准库。

## 数据库准备

脚本中使用的数据库和表会根据需要自动创建。如果你已有 MySQL 服务，请确保：

- MySQL 服务已启动，端口默认 3306
- 存在具有创建数据库/表权限的用户（例如 root）
- 根据脚本中的配置修改 `host`、`user`、`password`、`db` 参数（建议使用环境变量或配置文件）

## 文件详解

### 1. `mysql数据库连接.py` —— 基础连接与查询

**教学目的**：演示如何使用 `pymysql` 连接 MySQL，执行一条简单的 SELECT 查询并打印结果。

**代码要点**：
- 使用 `pymysql.connect()` 建立连接。
- 创建游标 (`cursor`) 执行 SQL。
- `cursor.fetchall()` 获取所有结果。
- 关闭游标和连接。

**运行**：
```bash
python mysql数据库连接.py
```

**注意事项**：
- 需提前创建数据库 `data` 和表 `product_info`，否则会报错。可先运行 `MySQL数据插入.py` 创建类似结构。
- 根据你的环境修改 `password` 等参数。

### 2. `MySQL数据插入.py` —— 创建库/表，单条插入

**教学目的**：
- 动态创建数据库（如果不存在）。
- 创建表（定义字段、主键、唯一约束）。
- 执行单条 INSERT 语句，并提交事务。

**代码流程**：
1. 连接 MySQL（指定数据库名，此处 `spider_data` 可能尚不存在，但连接时可不指定 `db` 参数，而是先创建库）。
2. 创建数据库 `spider_data`（如果不存在）。
3. 使用该数据库。
4. 创建表 `products`，其中 `url` 字段设为 `UNIQUE`。
5. 插入一条模拟数据。
6. 提交事务，关闭连接。

**运行**：
```bash
python MySQL数据插入.py
```

**教学要点**：
- `CREATE DATABASE IF NOT EXISTS` 保证幂等性。
- `CREATE TABLE IF NOT EXISTS` 同样避免重复创建报错。
- `url VARCHAR(500) UNIQUE` 为后续去重打下基础。
- 使用 `%s` 占位符而非字符串拼接，防止 SQL 注入。
- 务必调用 `conn.commit()` 持久化数据。

### 3. `MySQL批量插入和去重.py` —— 批量插入 + `INSERT IGNORE`

**教学目的**：
- 演示批量插入多条数据（`executemany`）。
- 利用 `INSERT IGNORE` 语法自动跳过重复记录（基于唯一键）。

**代码要点**：
- 连接数据库，假定 `products` 表已存在。
- 准备包含重复项的示例数据。
- `INSERT IGNORE INTO products ...`：当插入的 url 与表中已有 url 冲突时，忽略该条插入，不报错。
- `cursor.rowcount` 返回实际插入的行数。
- 提交事务。

**运行**：
```bash
python MySQL批量插入和去重.py
```

**输出示例**：
```
实际插入 3 条（重复的自动跳过）
```

**教学要点**：
- `INSERT IGNORE` 是一种简单的去重方式，适合对数据完整性要求不高的场景。
- 更严格的做法可使用 `INSERT ... ON DUPLICATE KEY UPDATE`。
- 批量插入能显著提高性能，尤其当数据量较大时。

### 4. `中关村笔记本数据.py` —— 完整异步爬虫 + 数据库存储

**教学目的**：
- 综合运用 `asyncio` + `aiohttp` 实现高并发爬虫。
- 学习异步请求的重试机制、信号量控制并发、随机 User-Agent。
- 使用 `parsel` 解析 HTML（CSS 选择器）。
- 将提取的笔记本参数通过 `aiomysql` 异步写入 MySQL。
- 使用 `uuid` 生成唯一编码作为去重依据。

**功能概述**：
- 爬取中关村在线笔记本频道的商品列表页。
- 从列表页提取单品详情页 URL。
- 从单品页中跳转到“参数”页（兼容两种页面结构）。
- 解析参数页中预定义的 50+ 项参数（如产品型号、价格、CPU、内存等）。
- 将参数数据写入 MySQL 表 `new_product_params`，每条数据生成一个 `unique_code`（UUID）并设置唯一索引，自动去重。
- 若产品型号为空，则跳过该条记录。

**代码结构**：
- `fetch_with_retry`：带指数退避重试的异步请求，随机 UA。
- `aioget_html_urls`：并发获取所有单品页 URL（使用信号量控制并发）。
- `aioget_html3_url_list`：从单品页提取参数页 URL（处理系列页和单品页两种结构）。
- `extract_params`：同步解析参数页，提取指定参数名对应的值。
- `aiorequests`：并发请求所有参数页，提取参数数据。
- `aiowriter`：异步写入 MySQL（使用 `aiomysql` 连接池），自动建表，通过 `unique_code` 去重。
- `main`：编排整个流程。

**运行**：
```bash
python 中关村笔记本数据.py
```

**输出**：日志会显示爬取进度、请求状态、插入数据条数，最后输出总耗时。

**教学要点**：
- **异步并发**：通过 `asyncio.gather` 并发执行大量请求，可显著提升爬取速度。
- **信号量**：`asyncio.Semaphore(100)` 限制同时进行的请求数，避免对目标网站造成过大压力。
- **重试机制**：指数退避（1s, 2s, 4s...）提高容错性。
- **随机 User-Agent**：每次请求前使用 `fake_useragent` 随机生成，降低被封风险。
- **异步数据库操作**：`aiomysql` 提供异步连接池，不会阻塞事件循环。
- **去重策略**：使用 `uuid.uuid4().hex` 生成唯一编码，并在数据库中设为 `UNIQUE KEY`，插入时自动避免重复。同时检查产品型号是否为空。
- **日志记录**：使用 `logging` 模块分级输出信息，便于调试。

**自定义修改**：
- 修改 `index_urls` 生成逻辑：目前只爬取第 1 页（`range(1,2)`），可增大页数获取更多数据。
- 修改 `param_names` 列表：可按需增删要提取的参数项。
- 修改 `db_config` 中的密码、数据库名等。

**注意事项**：
- 目标网站可能有反爬机制，请合理设置并发数（`Semaphore`）和重试策略。
- 爬取大量数据时建议增加延迟或使用代理。
- 请尊重 `robots.txt` 和网站服务条款，本示例仅用于教学目的。

## 教学建议顺序

1. **基础数据库操作**：先熟悉 `mysql数据库连接.py` 和 `MySQL数据插入.py`，理解连接、建表、增删改查。
2. **批量操作**：通过 `MySQL批量插入和去重.py` 学习批量插入和简单去重。
3. **异步爬虫入门**：如果尚未学习异步，可先参考目录中的 `aiohttp_简单使用.py` 和 `协程实战.py`（见前文异步示例集）。
4. **综合项目**：最后挑战 `中关村笔记本数据.py`，理解异步爬虫与数据库的完整结合。

## 常见问题

- **`pymysql.err.OperationalError: (1049, "Unknown database '...'")`**  
  检查数据库名是否正确，或先运行 `MySQL数据插入.py` 创建数据库。

- **`aiomysql` 连接失败**  
  确保 MySQL 服务已启动，且用户名、密码、主机配置正确。可尝试使用命令行客户端测试连接。

- **爬虫请求被拒绝（403, 418 等）**  
  增加随机延迟，使用更多样的 User-Agent，或引入代理 IP 池。

- **数据插入重复**  
  代码中已使用 `unique_code` 唯一索引。若要基于业务字段去重（如产品型号 + 屏幕尺寸），可修改表的唯一约束。

- **异步代码调试困难**  
  建议在 `fetch_with_retry` 中加入更多日志，或使用 `asyncio.run(main())` 时捕获异常。
