# Python 多线程与队列爬虫教学示例集

本目录包含多个演示 Python 多线程、线程同步、生产者-消费者模式以及使用队列进行任务调度的脚本。同时附带一个已爬取的 CSV 数据文件，展示了多线程爬虫的输出结果。所有代码均从实际教学场景出发，逐步深入，帮助理解并发编程的核心思想。

## 目录结构

```
.
├── chapters.csv                # 多线程爬虫的输出示例（小说章节数据）
├── lock.py                     # 线程锁（Lock）的基本使用
├── queue队列.py                # 生产者-消费者模式：多线程爬取章节并保存为 CSV
├── queue队列（优化）.py        # 优化版：多线程爬取并保存为独立 TXT 文件
├── threading_learn.py          # 线程的基础创建与启动
└── threading_more.py           # 多线程处理列表数据并收集结果
```

## 环境依赖

所有脚本基于 Python 3.6+ 编写，需安装以下第三方库：

```bash
pip install requests parsel fake-useragent
```

> 说明：`queue`、`threading`、`csv`、`os`、`time`、`random` 等均为 Python 标准库，无需额外安装。

## 文件详解

### 1. `threading_learn.py` —— 线程基础

**教学目的**：掌握如何创建并启动线程，理解 `join()` 的作用。

- 定义两个简单函数 `worker()`（含 `time.sleep(2)`）和 `T2()`（不含延迟）。
- 分别创建线程并启动，使用 `join()` 等待线程结束。
- 演示了如何设置线程名称、获取当前线程信息等辅助功能（注释部分）。

**运行**：
```bash
python threading_learn.py
```

**输出示例**：
```
T1 start
（等待2秒）
stop
T2 start
stop
done
```

**教学要点**：
- `threading.Thread(target=func, name="...")` 创建线程对象。
- `.start()` 启动线程，不会阻塞主线程。
- `.join()` 阻塞主线程，直到该线程执行完毕。
- 若不调用 `join()`，主线程可能先于子线程结束，导致看不到子线程的打印结果。

### 2. `threading_more.py` —— 多线程处理列表并收集结果

**教学目的**：演示多个线程如何并行处理数据，并通过队列收集各线程的返回结果。

- 定义一个二维列表 `a`（5 行，每行 5 个元素）。
- 每个线程独立处理一行，将每个元素平方。
- 处理完成后，将整行结果放入队列 `q`。
- 主线程等待所有子线程结束，然后从队列中取出所有结果并打印。

**运行**：
```bash
python threading_more.py
```

**输出**：
```
[[1, 1, 1, 1, 1],
 [4, 4, 4, 4, 4],
 [9, 9, 9, 9, 9],
 [16, 16, 16, 16, 16],
 [25, 25, 25, 25, 25]]
```

**教学要点**：
- 多线程可以同时处理不相关的数据切片（无竞争关系时）。
- 使用 `queue.Queue` 是线程安全的收集结果的方式。
- `join()` 确保所有线程完成后再继续。
- 此模式可推广到“分块处理”任务，如批量计算、文件处理等。

### 3. `lock.py` —— 线程同步：锁（Lock）

**教学目的**：理解多线程竞争共享资源的问题，掌握使用 `threading.Lock` 进行互斥访问。

- 定义全局变量 `A` 和锁对象 `lock`。
- 两个函数分别对 `A` 进行不同步长的累加（`worker1` 每次 +1，`worker2` 每次 +5）。
- 分别演示了 `acquire()/release()` 显式加锁和 `with lock` 上下文管理器两种方式。
- 最终输出 `A` 的最终值（应为 5×1 + 5×5 = 30）。

**运行**：
```bash
python lock.py
```

**可能输出**（顺序不定）：
```
worker1 A: 1
worker1 A: 2
worker2 A: 7
worker2 A: 12
...
worker2 A: 30
```

**教学要点**：
- 不加锁时，两个线程交错执行会导致数据错乱（例如同时读取 `A` 后分别写入，导致结果小于预期）。
- 锁保证了同一时刻只有一个线程能执行受保护的代码块。
- `with lock:` 语法更简洁安全，即使发生异常也会自动释放锁。

### 4. `queue队列.py` —— 生产者-消费者模式爬虫（CSV 输出）

**教学目的**：结合多线程、队列、爬虫，实现一个完整的并发下载器。

**功能**：
- 从指定小说目录页解析出所有章节 URL，放入 `url_queue`。
- 启动 5 个工作线程（消费者），不断从队列中获取 URL 进行请求、解析。
- 解析出的标题和正文内容放入 `result_queue`。
- 单独启动一个写入线程，从 `result_queue` 取出数据，写入 `chapters.csv` 文件。
- 使用 `set` 和锁实现 URL 去重。

**运行**：
```bash
python queue队列.py
```

**输出**：生成 `chapters.csv`，包含三列：`title`、`url`、`content`（content 为段落列表的字符串形式）。

**教学要点**：
- **生产者**：主线程负责将 URL 放入队列（实际本脚本中生产者在主线程一次性完成）。
- **消费者**：多个工作线程并发消费队列中的 URL。
- **结果收集队列**：避免多个线程同时写文件造成混乱。
- `queue.Queue()` 是线程安全的，无需额外加锁。
- `url_queue.task_done()` 和 `url_queue.join()` 配合使用，等待所有任务完成。
- 超时机制 `queue.get(timeout=5)` 防止线程无限阻塞。

**可改进点**：
- 实际生产者（获取 URL）也可以放到一个单独线程中，实现真正的异步生产。
- 异常处理不够精细，生产环境可增加重试机制。

### 5. `queue队列（优化）.py` —— 优化版：保存为独立 TXT 文件

**教学目的**：在上一版基础上，将每个章节保存为单独的 `.txt` 文件，并按书名创建文件夹。

**主要改进**：
- 使用 `get_url()` 函数从目录页提取章节 URL 和书名。
- `producer()` 将 URL 放入队列。
- `consumer()` 解析章节标题和正文，将结果放入 `result_queue`。
- `result_writer()` 从结果队列取数据，创建文件夹并写入 TXT 文件。
- 支持两种页面结构（`div#htmlContent p` 和 `div#text p`）。

**运行**：
```bash
python queue队列（优化）.py
```

**输出**：在 `D:\小说\<书名>` 目录下生成每个章节的单独 TXT 文件（文件名 = 章节标题）。

**教学要点**：
- 多线程爬取章节正文，并发写入不同文件，无需锁（因为每个线程写不同文件）。
- 使用 `os.makedirs` 自动创建多级目录。
- 队列的超时退出机制避免线程死锁。
- 清晰的代码分层：获取 URL、生产者、消费者、结果写入。

**注意事项**：
- 示例中写入路径硬编码为 `D:\小说\`，可根据需要修改为当前目录 `./小说`。
- 随机延迟（1~3 秒）可降低被封 IP 的风险，但在多线程下总延迟仍然较高，可适当减小。

1. **线程基础**：`threading_learn.py` → 理解创建、启动、等待。
2. **多线程计算**：`threading_more.py` → 理解多线程并行处理数据及结果收集。
3. **线程同步**：`lock.py` → 理解竞争条件和锁的必要性。
4. **队列与生产者-消费者**：`queue队列.py` → 结合爬虫，体会线程安全的任务分发。

6. **数据分析**：可让学生基于 `chapters.csv` 做文本处理或可视化。

## 常见问题

- **为什么我的多线程爬虫速度没有明显提升？**  
  可能受限于网络 I/O 或目标网站的反爬策略。可增加线程数（如 10~20）并适当减少随机延迟，同时注意礼貌爬取。

- **出现 `UnicodeEncodeError`**  
  确保文件写入时使用 `encoding='utf-8-sig'` 或 `utf-8`，并在 CSV 读写时指定同样编码。

- **队列中的任务永远执行不完**  
  检查是否调用了 `task_done()`，以及 `join()` 的位置是否正确。如果某个消费者异常退出，队列可能永远无法 `join`。

- **使用 `queue.get(timeout)` 后线程退出，但还有任务未完成**  
  确保超时时间大于每个任务的最大处理时间，或者采用更优雅的哨兵值（例如 `None`）来通知线程退出。

- **多个线程同时写一个文件导致混乱**  
  解决方案：使用队列 + 单写入线程（如本示例），或使用文件锁（`threading.Lock`）。
