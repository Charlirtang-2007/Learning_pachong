# Python Ajax 异步爬虫实战教学示例集

本目录包含三个渐进式的 Ajax 爬虫示例，演示如何从 **JSON API** 中异步获取数据并保存为文本文件。代码从同步分析接口规律开始，逐步过渡到高性能异步并发爬取，适合 Python 爬虫进阶学习者自学。

## 📁 目录结构

```
.
├── Ajax简单测试.py                    # 同步版：分析接口规律并测试下载
├── Ajax实战_ID 拼接型（详情页常见）.py  # 异步版：爬取百度阅读 API 的《西游记》
├── Ajax实战_简单翻页逻辑类型.py        # 异步版：爬取 apibi.cc API 的《九星霸体诀》
├── crawler.log                        # 异步爬虫运行日志（记录成功章节）
├── 第一回 灵根育孕源流出 心性修持大道生.txt   # 爬取结果示例
├── 第二回 悟彻菩提真妙理 断魔归本合元神.txt   # 爬取结果示例
└── 第三回 四海千山皆拱伏 九幽十类尽除名.txt   # 爬取结果示例
```

## 🚀 环境依赖

所有脚本需要 Python 3.7+，并安装以下第三方库：

```bash
pip install aiohttp aiofiles fake-useragent tenacity
```

> 说明：`json`、`asyncio`、`random`、`logging`、`os`、`urllib` 等为标准库，无需额外安装。

## 📖 文件详解

### 1. `Ajax简单测试.py` —— 同步接口分析与测试

**教学目的**：  
- 学习如何通过浏览器开发者工具（F12）分析 Ajax 请求规律。  
- 掌握如何从 API 响应中提取章节列表，并拼接出每一章的 URL。  
- 使用同步方式（`requests`）下载少量章节验证思路。

**功能**：  
- 以百度阅读的《西游记》为例，从目录 API 获取所有章节的 `cid`。  
- 动态构造章节内容 API 的 URL（通过拼接 `book_id` 和 `cid`）。  
- 遍历前 3 章，用 `requests.get` 下载内容并保存为 `.txt` 文件。

**运行**：
```bash
python Ajax简单测试.py
```

**输出**：当前目录下生成 `第一回 灵根育孕源流出 心性修持大道生.txt` 等文件（示例中前 3 章）。

**教学要点**：
- 抓包分析：打开浏览器开发者工具 → 网络 → XHR → 查看请求 URL 和参数。
- URL 构造规律：`https://dushu.baidu.com/api/pc/getCatalog?data={"book_id":"4306063500"}` 返回目录，其中每个章节有 `cid`；内容 API 为 `https://dushu.baidu.com/api/pc/getChapterContent?data={"book_id":"...","cid":"...|...","need_bookinfo":1}`。
- 同步请求的缺点：串行下载，速度慢，不适合大量章节。

### 2. `Ajax实战_ID 拼接型（详情页常见）.py` —— 异步版：百度阅读 API

**教学目的**：  
- 学习使用 `asyncio` + `aiohttp` 实现高并发异步请求。  
- 掌握自定义异步重试装饰器（基于 `tenacity`）。  
- 学习使用信号量（`Semaphore`）控制并发数，避免对目标服务器造成过大压力。  
- 完善日志记录（同时输出到控制台和文件）。  

**功能**：  
- 异步获取《西游记》全部章节目录（共 101 章）。  
- 异步并发下载所有章节内容，默认并发数 10（可调整）。  
- 使用 `aiofiles` 异步写入文件，按书名创建文件夹 `D:\小说\西游记`。  
- 自动重试网络异常（最多 3 次，指数退避）。  
- 所有请求随机延时 1~3 秒，模拟人类行为，降低被封风险。

**运行**：
```bash
python "Ajax实战_ID 拼接型（详情页常见）.py"
```

**输出**：
- 日志输出到控制台和 `crawler.log` 文件。  
- 章节文件保存在 `D:\小说\西游记\` 下，每个章节一个 `.txt` 文件（如 `第一回 灵根育孕源流出 心性修持大道生.txt`）。  
- 程序运行日志示例已在 `crawler.log` 中展示（101 章全部成功）。

**核心代码片段解析**：
```python
@retry_async(max_attempts=3)
async def get_titles(session, book_url, headers):
    # 获取目录，构造章节 URL 列表
    ...

sem = asyncio.Semaphore(10)   # 控制并发
tasks = [aiodownload(session, url, headers, title, name, sem) for ...]
await asyncio.gather(*tasks)
```

**教学要点**：
- **异步优势**：101 个章节在几秒内即可全部下载完成（同步需数十秒）。  
- **重试机制**：`tenacity` 库简化了重试逻辑，尤其适合网络请求。  
- **信号量**：防止同时发送过多请求导致 IP 被封。  
- **异步文件写入**：避免阻塞事件循环。

### 3. `Ajax实战_简单翻页逻辑类型.py` —— 异步版：apibi.cc API

**教学目的**：  
- 处理另一种 API 风格：分页参数通过 `params` 传递，章节内容通过 `chapterid` 索引。  
- 学习如何从第一个 API 获取 `title` 列表，然后根据索引构造第二个 API 的参数。  
- 对比“ID 拼接型”和“翻页逻辑型”两种 API 的差异。

**功能**：  
- 爬取 `apibi.cc` 网站的小说《九星霸体诀》（ID=1152）。  
- 先调用 `/api/booklist` 获取所有章节标题列表。  
- 再根据章节索引（`chapterid` 从 1 开始）构造 `/api/chapter` 请求，获取每一章的正文。  
- 异步并发下载，保存到 `D:\小说\九星霸体诀\` 目录。

**运行**：
```bash
python "Ajax实战_简单翻页逻辑类型.py"
```

**输出**：`D:\小说\九星霸体诀\` 下生成每个章节的 `.txt` 文件。

**教学要点**：
- 注意 API 响应结构：`/api/booklist` 返回 `{"list": ["第一章标题", "第二章标题", ...]}`。  
- 章节内容 API 需要 `id` 和 `chapterid` 两个参数，`chapterid` 与列表索引 +1 对应。  
- 异常处理：JSON 解析失败时打印前 200 字符，便于调试。  
- 内容字段：这里正文在 `data["txt"]` 中，标题在 `data["chaptername"]` 中。

## 🧪 自学路线建议

1. **理解 Ajax**：打开浏览器开发者工具，观察百度阅读或类似网站的 XHR 请求，找到真实的 API 接口。  
2. **同步试水**：运行 `Ajax简单测试.py`，修改参数尝试下载更多章节，理解 URL 构造逻辑。  
3. **异步入门**：运行 `Ajax实战_ID 拼接型（详情页常见）.py`，观察日志输出和运行时间，对比同步版本的差异。  
4. **举一反三**：修改 `Ajax实战_简单翻页逻辑类型.py` 中的 `id` 参数，尝试爬取其他小说（需先确认 API 是否开放）。  
5. **自主实践**：找一个类似的小说网站，抓包分析其 API，然后仿照本示例编写自己的异步爬虫。
